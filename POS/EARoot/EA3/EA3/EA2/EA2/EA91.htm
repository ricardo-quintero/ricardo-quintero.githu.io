<b><u>CU Breve:  </u></b>El Cajero registra las ventas del día y el Sistema calcula el total de la venta junto con sus impuestos. El Sistema, además, imprime el recibo que soporta la venta para el caso de devoluciones.<br/><b><u><br/></u></b><b><u>Scope:</u></b> Nextgen POS application<br/><b><u><br/></u></b><b><u>Stakeholders e intereses</u></b><b>:</b><br/><br/><ul>
<li><b>Cajero: </b>Desea una entrada de datos exacta y rápida, que no existan errores de pago y que en caso de pérdidas de efectivo le sean deducidas de su salario.</li><li><b>Vendedor:</b> Desea que sus comisones se actualicen.</li></ul>
<br/><br/><b><u>Trigger:</u></b> Este CU inicia cuando el cliente desea pagar los bienes y servicios en el POS<br/>

Steps:
1. El Cliente llega al POS con bienes y servicios a pagar.<br/>
2. El Cajero inicia una nueva venta.<br/>
3. El Cajero ingresa el código del producto.<br/>
4. El Sistema registra  la línea de detalle y presenta la descripción del producto, el precio y el total. El Precio se calcula a partir de ciertas reglas de negocio RN01-Se aplicará un descuento del 10% a todos los estudiantes del Tec;RN02-Se aplicará un descuento del 15% en compras mayores a 1000 pesos<br/>
5. El Sistema presenta el total con impuestos calculados.
<br/>
6. El Cajero le pide al cliente el total y le pide al cliente el pago.<br/>
7. El Cliente paga y el Sistema maneja el pago.<br/>
8. El Sistema registra la venta completa y envia la información de pago a los sistemas externos de Contabilidad e Inventarios.
<br/>
9. El Sistema muestra el recibo.
<br/>
10. El cliente sale  con el recibo y los bienes.
<br/>
